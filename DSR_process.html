만든 과정

Q. 이 웹페이지를 꾸밀 때 어떤 목표를 갖고 시작했나요?

자기 소개 페이지는 저를 한눈에 파악할 수 있게 만들려고 노력했습니다. 또한 필수 내용인 저의 소개와 계획이 다른 내용보다 잘 보일 수 있도록 하였습니다.


Q. 목표를 이루기 위한 진행 과정에서 어려운 점을 어떻게 극복했나요?

내용 구성에 대해 생각하다보니, UI 구성과 애니메이션을 넣을 시간이 없을 것 같았습니다.
그래서 들어가야할 내용을 먼저 넣고 시간이 남으면 다듬기 위해, html을 작성할 때 class와 id를 직관적으로 작성하려고 노력했습니다.
이 과정에서 class와 id의 개념이 헷갈려서 고난을 겪었습니다. 그래서 예전에 블로그에 직접 정리해놓은 것을 바탕으로,
class는 중복 사용이 가능하고, id는 한 번만 사용 가능하다는 것을 알고 저의 소개 페이지를 완성할 수 있었습니다. 추후 보강할 기회가 생긴다면 정리해놓은 것을 바탕으로 수정하고 싶습니다.


Q. 완성된 웹페이지를 볼 때 목표의 몇 %가 구현되었다고 생각하나요?

70% 정도 구현한 것 같습니다.


Q. 시간이 더 주어진다면 어떤 점을 더 보강하고 싶은가요?

먼저 기능 면에서는, Navigation bar을 이용해서 '저를 소개합니다'/'학습계획'으로 스크롤이 자동으로 이동하는 기능을 구현하고 싶습니다.

시각적으로 보강하고 싶은 것은,
전체적으로 폰트를 사용하여 시각적인 즐거움을 더하고, 제 이름인 '도소라'라는 하이퍼링크에 hover이벤트를 추가하고 싶습니다.

내용 구성에서 보강하고 싶은 것은, 가능한 프로그래밍 언어 목록을 Progress Bar로 보여주고 싶습니다.
